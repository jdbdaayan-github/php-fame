PHP Fame Framework Documentation

1. Installation

Copy the project files to your server. Set public/ as the web root. Run composer install if using dependencies.

2. Project Structure

project-root/
├── app/                  # Controllers, Models, Views, Services
├── bootstrap/            # Core bootstrap
├── config/               # Config files (app.php, cache.php, database.php, mail.php, queue.php, services.php)
├── public/               # Web root, index.php entry
├── routes/               # web.php, api.php
├── storage/              # Logs, cache, sessions
├── system/               # Core framework: Http, Security, Database, Support
├── tests/                # Unit & Feature tests
├── vendor/               # Composer dependencies
├── .env                  # Environment variables
└── README.md             # Project info

3. Application

The System\Application class initializes the framework and kernel.

$app = new System\Application();
$app->run();

4. Kernel

Handles all requests and dispatches to the router.

5. Router

Define routes in routes/web.php:

Route::get('/', [HomeController::class, 'index']);
Route::post('/user', [UserController::class, 'store']);

6. Request & Response

Request handles method, URI, input. Response handles content, headers, and status.

7. Controllers

Controllers handle business logic and return Response or view().

8. Views

Templates use *.fame.php. Example:

view('home', ['name' => 'JDBD']);

9. Security

  • Authentication (Auth)
  • CSRF Protection (@csrf)
  • Password Hashing (bcrypt/argon2)
  • Encryption/Decryption
  • Token Management (JWT/API)
  • Input Validation
  • Rate Limiting

10. Database

Configure in config/database.php or via .env. Supports MySQL, PostgreSQL, SQLite, SQL Server, Oracle.

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=fame_db
DB_USERNAME=root
DB_PASSWORD=

Use System\Database\DB and QueryBuilder for queries.

11. Facades

Static access to core components, e.g., Route::get(...).

12. Middleware

Request filtering: authentication, CORS, logging, CSRF, etc.

13. Environment

Use .env or config files. Example:

APP_NAME=Fame
APP_ENV=local
APP_URL=http://localhost
APP_DEBUG=true

14. Helpers

Common functions:

  • base_path()
  • view()
  • config()
  • redirect()
  • url()

15. Themes

  • Dark sidebar
  • Light content area
  • Customizable layout

16. Fame Templating Engine

Blade-like syntax with layouts, sections, includes, variables, control structures, CSRF, and method spoofing.

@include('partials.header')
@extends('layout')
@section('content') ... @endsection
@if($users) ... @endif
@csrf
@method('PUT')

17. Error Pages

Custom 404, 403, 500 pages are in app/Views/errors/:

app/Views/errors/404.fame.php
app/Views/errors/403.fame.php
app/Views/errors/500.fame.php

Example 404 page:

<h1>404 Not Found</h1>
<p>The page you requested could not be found.</p>
<a href="<?= url('/') ?>">Go Home</a>
